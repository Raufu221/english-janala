Frequently Asked Questions

1. What is the difference between var, let, and const?

var is the old way. It is function-scoped and can be re-declared, which can cause bugs.
let is the new way for variables that can change. It is block-scoped (stays inside its `{ }`) and cannot be re-declared.
const is for variables that *cannot* change (constants). It is also block-scoped and cannot be re-declared or updated.

2. What is the Difference between map(), forEach(), and filter()?

forEach(): This just executes a function for each item in the array. It does not return anything. It's used when you want to do something (like print to the console or update a database) for each item. map(): This transforms an array. It executes a function for each item and returns a new array of the same length with the results. (e.g., [1, 2, 3] -> [2, 4, 6])
filter(): t This selects items from an array. It executes a function that returns true or false for each item and returns a new array containing only the items that passed the test (returned true).

3. Arrow Functions vs. Regular Functions

Regular Functions: (e.g., function() { ... }) create their own this context. The value of this depends on how the function is called (e.g., as a method of an object, as a standalone function, etc.).
Arrow Functions: (e.g., () => { ... }) do not have their own this. They inherit the this value from their surrounding (lexical) scope. This makes them much more predictable, especially inside loops or callback functions. They are also shorter to write.
